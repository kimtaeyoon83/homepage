# LLM 파인튜닝, 그리고 LoRA — 과거부터 현재까지의 흐름

대형 언어 모델을 다루다 보면 "파인튜닝"이라는 단어를 매일 마주치지만, 그 안에서 무엇이 어떻게 바뀌어 왔는지는 의외로 흐릿하다. 이 글에서는 파인튜닝의 기본 개념부터 LoRA의 작동 원리, 그리고 그 이후의 진화까지를 하나의 흐름으로 정리한다.

## 파인튜닝이란

사전학습(pre-training)으로 일반적인 언어 능력을 갖춘 모델을, 특정 작업·도메인·말투에 맞게 가중치를 추가 학습시키는 과정이다. 핵심 질문은 언제나 하나다.

> **전체 파라미터 중 무엇을, 얼마나 건드릴 것인가.**

파인튜닝 기법의 역사는 사실상 이 질문에 대한 답이 점점 정교해진 역사다.

### 1) Full Fine-tuning (2018~2019)

모델의 **모든 파라미터**를 업데이트하는 방식이다. 성능은 가장 확실하지만, 작업마다 모델 전체 사본(수십~수백 GB)을 따로 저장해야 하고, 옵티마이저 상태(Adam은 파라미터당 2배 추가 메모리)까지 GPU에 올려야 한다. GPT-3(175B)급이 등장하면서 일반 연구자·기업에게는 사실상 비현실적이 됐다.

### 2) PEFT의 등장 (2019~2021)

"대부분을 얼리고(freeze) 일부만 학습하자"는 발상, 즉 Parameter-Efficient Fine-Tuning이다. 초기 시도들은 다음과 같다.

- **Adapter** (Houlsby, 2019): 트랜스포머 레이어 사이에 작은 병목 모듈을 삽입. 단, 레이어가 늘어나니 추론 시 latency가 증가하는 단점이 있다.
- **Prefix / Prompt Tuning** (2021): 가중치는 그대로 두고, 입력 앞에 학습 가능한 가상 토큰만 붙인다. 가볍지만 표현력에 한계가 있다.
- **BitFit**: bias 항만 학습. 극단적으로 가볍지만 성능 제약이 크다.

### 3) LoRA (2021) — 게임 체인저

Adapter의 "추론 latency" 문제를 우아하게 해결했다. 아래에서 자세히 다룬다.

### 4) QLoRA (2023) — 접근성 폭발

베이스 모델을 **4-bit로 양자화**해서 메모리에 올리고, 그 위에 LoRA 어댑터만 학습한다. 단일 48GB GPU로 65B 모델 파인튜닝을 가능하게 만든 분기점이다. 세 가지 핵심 기술이 있다.

- **NF4 (NormalFloat 4-bit)**: 정규분포를 따르는 가중치에 최적화된 4-bit 데이터 타입
- **Double Quantization**: 양자화 상수 자체를 또 양자화해 메모리를 추가 절감
- **Paged Optimizer**: 메모리 스파이크 시 옵티마이저 상태를 CPU로 페이징

## LoRA는 어떤 방식인가

LoRA(Low-Rank Adaptation)의 출발점은 하나의 관찰이다. **파인튜닝으로 생기는 가중치 변화(ΔW)는 사실 매우 낮은 랭크(low-rank)를 가진다.** 즉, 거대한 ΔW 행렬을 통째로 학습할 필요 없이, 두 개의 훨씬 작은 행렬의 곱으로 근사할 수 있다는 것이다.

![LoRA 구조: 입력 x가 고정된 사전학습 가중치 W0 경로와, 저랭크 행렬 A·B로 분해된 학습 경로로 나뉘어 흐른 뒤 합산되어 출력 h가 된다](images/lora-structure.png)

원래 forward 연산은 `h = W₀·x` 다. LoRA는 여기에 학습 가능한 경로를 더한다.

```
h = W₀·x + ΔW·x = W₀·x + (α/r)·B·A·x
```

각 구성 요소는 다음과 같다.

- **W₀ (d×k)** — 사전학습 가중치. 완전히 고정(freeze)한다. 그래디언트가 흐르지 않는다.
- **A (r×k)** — 입력을 `r`차원으로 압축. 가우시안 랜덤으로 초기화한다.
- **B (d×r)** — 다시 `d`차원으로 복원. **0으로 초기화**한다. 따라서 학습 시작 시점에 `ΔW = BA = 0`이라, 베이스 모델 성능을 그대로 유지한 채 출발한다.
- **r (rank)** — 보통 4~64. `r << min(d, k)`. LoRA의 핵심 하이퍼파라미터다.
- **α/r (스케일링)** — 학습률처럼 작동하는 스케일 계수다.

### 파라미터 절감 효과

핵심은 학습 가능한 파라미터가 극적으로 줄어든다는 점이다. 예를 들어 `d = k = 4096`이라면,

| 방식 | 학습 파라미터 수 | 비율 |
|------|------------------|------|
| Full Fine-tuning | 4096 × 4096 ≈ **1,680만** | 100% |
| LoRA (r=8) | (4096×8) + (8×4096) ≈ **6.5만** | **약 0.4%** |

99% 이상이 줄어든다. 그러면서도 Adapter와 달리 **추론 시 latency가 0**인데, 이유는 학습이 끝나면 `W = W₀ + BA`로 **병합(merge)**할 수 있기 때문이다. 병합하면 구조적으로 원래 모델과 완전히 동일해진다.

여기서 우아한 운영상의 이점이 나온다. 베이스 모델은 하나만 메모리에 올려두고, 작업별 LoRA 어댑터(수십 MB)만 갈아끼우면 된다. 이것이 바로 단일 GPU에서 수천 개의 LoRA 어댑터를 동시 서빙하는 **S-LoRA**가 가능한 이유다.

## 현재 흐름 (2024~2025)

LoRA 이후의 진화는 "low-rank 근사를 어떻게 더 정교하게 할까"로 수렴한다.

- **DoRA** (Weight-Decomposed LoRA, 2024): 가중치를 **크기(magnitude)**와 **방향(direction)**으로 분해하고, 방향에만 LoRA를 적용한다. Full FT에 더 가까운 학습 패턴을 보여, 같은 rank에서 성능이 더 높다.
- **LoRA+**: A와 B에 서로 다른 학습률을 적용(B를 더 크게)해 수렴 속도와 성능을 개선한다.
- **rsLoRA**: 스케일링을 `α/r` 대신 `α/√r`로 바꿔 높은 rank에서도 안정적으로 학습한다.
- **VeRA**: A, B를 공유 랜덤 행렬로 고정하고 작은 스케일링 벡터만 학습해, 파라미터를 더 극단적으로 절감한다.
- **QA-LoRA / QLoRA 계열**: 양자화와 LoRA의 결합을 더 매끄럽게 만들고, AWQ·GPTQ 같은 추론용 양자화와의 호환성에 초점을 둔다.

## 정리

큰 그림으로 보면 흐름은 이렇게 정리된다.

> **전부 학습(Full FT)** → **일부만 학습(Adapter / Prefix)** → **저랭크로 분해해 학습(LoRA)** → **양자화로 메모리까지 압축(QLoRA)** → **분해 방식 자체를 정교화(DoRA 등) + 대규모 멀티 어댑터 서빙(S-LoRA)**

실무 선택 기준만 짧게 덧붙이면 이렇다. 도메인 적응·말투 학습 정도면 LoRA로 충분하고, GPU 메모리가 빠듯한 대형 모델이면 QLoRA, 같은 자원에서 품질을 더 끌어올리고 싶으면 DoRA를 검토하는 흐름이 일반적이다.
